# Compression Guide

Use this guide when turning source material into a poster plan.

## Core Brief

Think like an Apple designer, a top poster designer, and a Takahashi-method presenter. Transform one or more long texts into a visual explanation that can be understood in 30-45 seconds.

Do not copy the article mechanically. Understand it, compress it, and present it by cognition order:
1. What is the central claim?
2. Why should the reader care?
3. What are the key points or evidence?
4. What changes in the reader's understanding after seeing it?
5. What single phrase should remain in memory?

## Reading Budget

Default single-poster budget:
- Title: 8-18 Chinese characters, or 3-8 English words.
- Core claim: 1 strong sentence.
- Sections: 3-4 groups.
- Bullets: 1-3 per group.
- Bullet length: short phrase or one compact sentence.
- Total visible body copy: usually under 220 Chinese characters or 120 English words.

If the source cannot fit this budget without losing the core meaning, create multiple posters. Never solve overload by shrinking body text until it becomes unreadable.

## Distillation Rules

- Keep the author's strongest idea, not the author's full wording.
- Prefer concrete nouns, verbs, numbers, contrasts, and consequences.
- Remove throat-clearing, examples that do not change the conclusion, repeated claims, and generic transitions.
- If the text is argumentative, show claim -> reason -> consequence -> boundary.
- If the text is instructional, show goal -> steps -> pitfalls -> result.
- If the text is narrative, show trigger -> turn -> insight -> takeaway.
- If multiple sources disagree, make the contrast visible instead of flattening them.
- If facts are time-sensitive, keep source/date context or verify with current sources before stating as fact.

## Visual Direction

- Design for phone readability at default preview size.
- Use oversized type for the core point.
- Build clear hierarchy: title, core claim, grouped details, final memory line.
- Let color, spacing, and typography clarify roles rather than decorate empty space.
- Choose palette and structure from the topic: serious, urgent, reflective, technical, emotional, practical, or editorial.
- Avoid decorative English labels, signatures, watermarks, AI labels, and process explanations unless the user explicitly asks.

## Output Strategy

Choose one of these structures:
- **One-shot insight**: huge title + core claim + 3 supporting blocks.
- **Framework**: title + principle + numbered groups + final line.
- **Contrast**: before/after, myth/fact, problem/solution.
- **Process**: sequence with 3-5 steps and one caution.
- **Batch**: one poster per source or per theme, wrapped in a folder.

Stop when the poster has one obvious first glance, one clean reading path, and one memorable conclusion.
